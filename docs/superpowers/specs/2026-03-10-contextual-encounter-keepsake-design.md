# Contextual Encounter & Keepsake Summary — Design Spec
**Date:** 2026-03-10
**App:** Halfway (`/Users/jeeminhan/code/global-atlas`)

---

## Overview

Redesign the Halfway encounter flow to:
1. Collect richer context (shared setting + per-person occupation) before the conversation begins
2. Replace per-round text input with a single ambient voice recording of the whole conversation
3. Replace the "halfway question" with a rich **keepsake artifact** — a saved card both people will want to keep and return to

---

## New Flow

```
setting → who-you → who-you-occupation → who-them → who-them-occupation → loading-topics → recording → processing → keepsake
```

### Removed steps
- `round` (per-round textarea input)
- `generating` (halfway question generation)
- `halfway` (halfway question display)

### Removed components
- `HalfwayQuestion.jsx` (replaced by `KeepsakeSummary.jsx`)
- `VoiceButton.jsx` (replaced by `RecordingScreen.jsx`)

### Removed API endpoints
- `api/halfway-question.js` (replaced by `api/summarize-conversation.js`)

---

## New State in EncounterFlow

```js
setting: string          // e.g. "campus" — shared
person1.occupation: string   // e.g. "student"
person2.occupation: string   // e.g. "professional"
topics: Topic[]          // AI-generated directional topic pairs
audioTranscript: string  // full transcript from recording
keepsake: Keepsake       // final artifact object
```

## New Components

### `SettingPicker.jsx`
- **Heading:** "Where are you?"
- **Subheading:** subtle, e.g. "This shapes what you'll ask each other."
- **UI:** staggered card grid (Framer Motion, terracotta accent), 8 options:
  - 🏫 Campus · ☕ Café · 📚 Library · ⛪ Church · 🌳 Park · 🏢 Office · 🏠 Home · 🏥 Hospital
- **Interaction:** tap to select (scale up, terracotta border glow), "Continue →" button appears
- **One selection required.** Collected once, shared for both people.

### `OccupationPicker.jsx`
- **Heading:** "What does [label] do?" where `[label]` = person's city if available, else country, else "this person". For `who-you-occupation`, Person 1's city/country is already in state (set during `who-you`). For `who-them-occupation`, Person 2's city/country is already in state (set during `who-them`). The fallback "this person" should never be needed in normal flow.
- **UI:** same staggered card grid; terracotta accent for Person 1, sage accent for Person 2
- **Options (8):**
  - 🎓 Student · 🔬 Researcher · 💼 Professional · 🤝 Volunteer · ✝️ Ministry · 🍳 Service · 👩‍💻 Tech · 🎨 Creative
- **Interaction:** same tap-to-select pattern. One selection required.

### `RecordingScreen.jsx`
- Replaces all `round` step UI
- **Shows:** the 3 AI-generated topic pairs as conversation guides — each topic card shows `question1` (for Person 1) and `question2` (for Person 2), both visible so each person knows what the other was asked
- **UI:** large pulsing mic button to start/stop; subtle waveform animation while recording; elapsed time counter
- **Max duration:** 5 minutes. At 4:00 a gentle warning appears ("wrapping up soon"); at 5:00 recording auto-stops
- **MediaRecorder API** captures audio as a blob (webm or ogg, browser default)
- "Finish Conversation →" ends recording and triggers processing
- **Back navigation:** shows a confirmation dialog ("Discard this recording?") before going back
- **Browser compatibility:** check `MediaRecorder` availability on mount. If unavailable (e.g. older iOS Safari), automatically show the fallback textarea with a brief note: "Voice recording isn't available on this browser. Type a summary of your conversation instead."

### `KeepsakeSummary.jsx`
- Displays the final keepsake artifact — a beautiful card saved to conversation history
- **Four sections:**
  1. **The Thread** — the specific shared thing named beautifully (1–2 sentences)
  2. **Two Windows** — what each person gave the other; one insight per person, labeled by city
  3. **A Closing Reflection** — poetic, gospel-threaded; names the longing underneath without asking a question
  4. **A Continue Prompt** — a short, warm, non-interactive suggestion for a human next step generated by AI (e.g. "Exchange numbers. You're not done."). Display only — not a button.
- **Share button:** attempts `navigator.share()` (Web Share API). Falls back to `navigator.clipboard.writeText()` + a "Copied!" toast if Web Share is unavailable (e.g. desktop browsers)
- **Save timing:** keepsake is saved to localStorage immediately after successful API response, before navigating to the keepsake screen. If navigation to keepsake screen is interrupted, the save is already complete.

### `ProcessingScreen` (inline in EncounterFlow, not a separate component)
- Shown during `processing` step while awaiting summarize-conversation API
- Animated loading indicator + copy: "Finding your halfway point…"
- **Error handling:** if API fails or times out (10s), show an error message with a "Try again" button (retries the API call once). If retry also fails, show a minimal fallback keepsake: `{ thread: "Something happened here that words can't quite hold.", person1Window: "", person2Window: "", reflection: "What if the thing you're both homesick for isn't actually a place?", continuePrompt: "Stay in touch." }`. Save this fallback keepsake to history and allow user to continue.
- Processing step is not cancellable (back button disabled)

## API Changes

### `api/generate-topics.js` — enriched prompt

**New inputs:** `setting`, `person1Occupation`, `person2Occupation` (added to existing city/country fields)

**Prompt changes:**
- Each topic now generates **two directional questions** — one question for Person 1 (to help them understand Person 2's world) and one for Person 2 (to help them understand Person 1's world)
- Questions use all five context signals: city, country, setting, and occupation
- Questions probe toward **longing, permanence, being fully known** — the surface is cultural, the underneath quietly points toward the hunger the keepsake will name

**New response shape:** Always exactly 3 objects (one per drawn topic, matching the existing `TOPICS_PER_GAME = 3` constant):
```json
[
  {
    "id": "loss",
    "question1": "...(for Person 1, to help them understand Person 2's world)",
    "question2": "...(for Person 2, to help them understand Person 1's world)"
  },
  ...
]
```
If the API returns fewer than 3 items, fall back to the original `topics` array's default questions.

### `api/summarize-conversation.js` — new endpoint (replaces `halfway-question.js`)

**Inputs:**
```json
{
  "transcript": "full audio transcript text",
  "setting": "campus",
  "person1": { "city": "Seoul", "country": "South Korea", "occupation": "student" },
  "person2": { "city": "Lagos", "country": "Nigeria", "occupation": "student" }
}
```

**Prompt:** Find the specific shared thread across both people's words. Generate:
- `thread` — the one thing that appeared in both worlds, named with their actual words
- `person1Window` — one beautiful cultural insight Person 2 learned about Person 1's world
- `person2Window` — one beautiful cultural insight Person 1 learned about Person 2's world
- `reflection` — a short poetic closing that names the longing underneath and quietly points toward a Person who could meet it fully — warm, specific, not preachy
- `continuePrompt` — a single, warm, human next step suggestion (1–2 sentences) encouraging the two people to stay connected or deepen the encounter. E.g. "Exchange numbers. You're not done." or "Ask them what they miss most about home." Not a question — a nudge.

**Response:**
```json
{
  "thread": "...",
  "person1Window": "...",
  "person2Window": "...",
  "reflection": "...",
  "continuePrompt": "..."
}
```

## RecordingScreen — Audio Pipeline

**Audio path (MediaRecorder available):**
1. `MediaRecorder` captures audio stream from `getUserMedia({ audio: true })`, browser-default format. Chrome/Edge produce `audio/webm`, Safari produces `audio/mp4`. Both are supported by Gemini.
2. On "Finish" → audio blob assembled from chunks
3. **Size check:** 5 minutes of voice audio at typical MediaRecorder bitrates (~32–64 kbps) produces ~1–2 MB, well within Gemini's 20 MB inline data limit. No compression step needed.
4. Blob converted to base64 and sent to `/api/summarize-conversation` as `audioBase64` (base64 string) + `audioMimeType` (e.g. `"audio/webm"` or `"audio/mp4"`)
5. API uses Gemini's inline audio input (`inlineData: { mimeType, data }`) — compatible with gemini-2.0-flash audio understanding
6. Gemini transcribes + summarizes → returns keepsake JSON

**Text fallback path (MediaRecorder unavailable):**
1. User types a summary of their conversation into a textarea
2. Sent to `/api/summarize-conversation` as `transcript` (string), `audioBase64` omitted
3. API detects text path (no `audioBase64`), uses a text-only Gemini prompt — same output shape

## Gospel Presence

- **In topic questions:** subtle thread toward longing, permanence, being fully known — cultural surface, deeper underneath
- **In the reflection:** names the hunger directly but gently, points toward a Person (not a place, not a feeling) who could meet it — same register as the existing `halfway-question.js` prompt
- Never preachy. Always earned by what the people actually said.

---

## Data Model — Conversation History

Saved to `localStorage` via existing conversation history mechanism (same key/structure as current app, extended):

```js
{
  id: "convo-...",
  person1: { city, country, occupation },
  person2: { city, country, occupation },
  setting: "campus",
  topics: [...],           // directional topic pairs shown during recording
  // transcript NOT saved — privacy-sensitive, used only for generation
  keepsake: {
    thread: "...",
    person1Window: "...",
    person2Window: "...",
    reflection: "...",
    continuePrompt: "..."
  },
  createdAt: "..."
}
```

## Back Navigation

| Step | Back behavior |
|------|--------------|
| `setting` | onClose (exits encounter) |
| `who-you` | `setting` |
| `who-you-occupation` | `who-you` |
| `who-them` | `who-you-occupation` |
| `who-them-occupation` | `who-them` |
| `loading-topics` | disabled (back button hidden, system gestures ignored) |
| `recording` | confirmation dialog → `who-them-occupation` (discards recording) |
| `processing` | disabled |
| `keepsake` | onClose (conversation already saved) |

## Implementation Order

1. `SettingPicker.jsx` + wire into EncounterFlow
2. `OccupationPicker.jsx` + wire into EncounterFlow (both persons)
3. `api/generate-topics.js` — enrich prompt with new context + directional questions
4. `RecordingScreen.jsx` — MediaRecorder + topic display
5. `api/summarize-conversation.js` — new endpoint with Gemini audio + keepsake prompt
6. `KeepsakeSummary.jsx` — keepsake artifact UI
7. Update conversation history data model
8. Remove `VoiceButton.jsx`, `HalfwayQuestion.jsx`, `api/halfway-question.js` — check all imports first; these files should have no remaining importers after steps 1–7

---

## Out of Scope
- Backend persistence (keepsake stays in localStorage like existing conversations)
- Authentication
- Sharing to external platforms beyond native share API
- Transcript storage (privacy-sensitive; transcript used only for generation, not saved by default)
